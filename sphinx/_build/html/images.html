OneAndOneService Methods¶

Note

All of the methods below can be called on a OneAndOneService instance.

list_images(page=None, per_page=None, sort=None, q=None, fields=None)¶

Returns a list of all images.

Parameters:
  • page (int) – Allows the use of pagination. Indicate which page to start on.
  • per_page (int) – Number of items per page.
  • sort (str) – sort='name' retrieves a list of elements sorted alphabetically. sort='creation_date' retrieves a list of elements sorted by their creation date in descending order.
  • q (str) – q is for query. Use this parameter to return only the items that match your search query.
  • fields (str) – Returns only the parameters requested. (i.e. fields=’id, name, description, hardware.ram’)
Return type:

JSON

The “Image” Class¶

class Image(server_id=None, name=None, description=None, frequency=None, num_images=None)¶

Pass an Image instance into the create_image() method to create an image. There are also a few helper methods available to perform simple requests after creating your image.

Parameters:
  • server_id (str) – the ID of the server to be copied.
  • name (str) – image name.
  • description (str) – image description.
  • frequency (str) – the image’s creation policy. Possible values are 'ONCE', 'DAILY', and 'WEEKLY'.
  • num_images (int) – maximum number of images.

Methods:

get()¶

Retrieves the image’s current specs.

wait_for()¶

Polls the Image resource until an 'ACTIVE', 'POWERED_ON', or 'POWERED_OFF' state is returned.
