list_shared_storages(page=None, per_page=None, sort=None, q=None, fields=None)¶

Returns a list of all shared storages.

Parameters:
  • page (int) – Allows the use of pagination. Indicate which page to start on.
  • per_page (int) – Number of items per page.
  • sort (str) – sort='name' retrieves a list of elements sorted alphabetically. sort='creation_date' retrieves a list of elements sorted by their creation date in descending order.
  • q (str) – q is for query. Use this parameter to return only the items that match your search query.
  • fields (str) – Returns only the parameters requested. (i.e. fields=’id, name, description, hardware.ram’)

The “SharedStorage” Class¶

class SharedStorage(name=None, description=None, size=None)¶

Pass a SharedStorage instance into the create_shared_storage() method to create a shared storage. There are also a few helper methods available to perform simple requests after creating your shared storage.

Parameters:
  • name (str) – shared storage name.
  • description (str) – shared storage description.
  • size (int) – shared storage size. Must be a multiple of 50.

Methods:

get()¶

Retrieves the shared storage’s current specs.

servers()¶

Returns a list of the servers attached to the shared storage.

wait_for()¶

Polls the SharedStorage resource until an 'ACTIVE', 'POWERED_ON', or 'POWERED_OFF' state is returned.

The “AttachServer” Class¶

class AttachServer(server_id=None, rights=None, server_ip_id=None)¶

Use the AttachServer class to attach servers or server IPs to various containers.

Parameters:
  • server_id (str) – the unique identifier for the server.
  • rights (str) – server rights for accessing a shared storage. Possible values are 'R' or 'RW'. This parameter is only needed when attaching servers to a shared storage.
  • server_ip_id (str) – the unique identifier for the server’s IP.
