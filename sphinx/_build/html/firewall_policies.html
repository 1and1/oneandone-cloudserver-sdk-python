The “FirewallPolicy” Class¶

class FirewallPolicy(name=None, description=None)¶

Pass a FirewallPolicy instance into the create_firewall_policy() method to create a firewall policy. There are also a few helper methods available to perform simple requests after creating your firewall policy.

Parameters:
  • name (str) – firewall policy name.
  • description (str) – firewall policy description.

Methods:

get()¶

Retrieves the firewall policy’s current specs.

ips()¶

Returns a list of the server IPs attached to the firewall.

rules()¶

Returns a list of the rules attached to the firewall.

wait_for()¶

Polls the FirewallPolicy resource until an ACTIVE, POWERED_ON, or POWERED_OFF state is returned.

The “FirewallPolicyRule” Class¶

class FirewallPolicyRule(protocol=None, port_from=None, port_to=None, source=None)¶

Use the FirewallPolicyRule class to create or modify firewall policy rules.

Parameters:
  • protocol (str) – internet protocol. Possible values are "TCP", "UDP", "ICMP", "AH", "ESP", "GRE".
  • port_from (int) – first port in range.
  • port_to (int) – second port in range.
  • source (str) – IPs from which access is available. Setting 0.0.0.0 all IPs are allowed.

The “AttachServer” Class¶

class AttachServer(server_id=None, rights=None, server_ip_id=None)¶

Use the AttachServer class to attach servers or server IPs to various containers.

Parameters:
  • server_id (str) – the unique identifier for the server.
  • rights (str) – server rights for accessing a shared storage. Possible values are 'R' or 'RW'.
  • server_ip_id (str) – the unique identifier for the server’s IP.
