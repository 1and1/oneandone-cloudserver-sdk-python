The “MonitoringPolicy” Class¶

class MonitoringPolicy(name=None, description=None, email=None, agent=None)¶

Pass a MonitoringPolicy instance into the create_monitoring_policy() method to create a monitoring policy. There are also a few helper methods available to perform simple requests after creating your monitoring policy.

Parameters:
  • name (str) – monitoring policy name.
  • description (str) – monitoring policy description.
  • email (str) – user’s email address.
  • agent (bool) – Set to True to use an agent.

Methods:

get()¶

Retrieves the monitoring policy’s current specs.

ports()¶

Retrieves a list of the ports of the monitoring policy.

processes()¶

Retrieves a list of the processes of the monitoring policy.

servers()¶

Retrieves a list of the servers currently attached to the monitoring policy.

wait_for()¶

Polls the MonitoringPolicy resource until an ACTIVE, POWERED_ON, or POWERED_OFF state is returned.

The “Threshold” Class¶

class Threshold(entity=None, warning_value=None, warning_alert=None, critical_value=None, critical_alert=None)¶

Use the Threshold class to add thresholds to a monitoring policy.

Parameters:
  • entity (str) – threshold name. Possible values are 'cpu', 'ram', 'disk', 'transfer', 'internal_ping'.
  • warning_value (int) – advise when this value is exceeded. (%)
  • warning_alert (bool) – enable or disable alerts.
  • critical_value (int) – advise when this value is exceeded. (%)
  • critical_alert (bool) – enable or disable alerts.

The “Port” Class¶

class Port(protocol=None, port=None, alert_if=None, email_notification=None)¶

Use the Port class to add ports to a monitoring policy.

Parameters:
  • protocol (str) – internet protocol. Possible values are 'TCP' or 'UDP'.
  • port (int) – port number.
  • alert_if (str) – case of alert. Possible values are 'RESPONDING' or 'NOT_RESPONDING'.
  • email_notification (bool) – elect to receive email notifications in case of alert.

The “Process” Class¶

class Process(process=None, alert_if=None, email_notification=None)¶

Use the Process class to add processes to a monitoring policy.

Parameters:
  • process (str) – process name.
  • alert_if (str) – case of alert. Possible values are 'RUNNING' or 'NOT_RUNNING'.
  • email_notification (bool) – elect to receive email notifications in case of alert.

The “AttachServer” Class¶

class AttachServer(server_id=None, rights=None, server_ip_id=None)¶

Use the AttachServer class to attach servers or server IPs to various containers.

Parameters:
  • server_id (str) – the unique identifier for the server.
  • rights (str) – server rights for accessing a shared storage. Possible values are 'R' or 'RW'.
  • server_ip_id (str) – the unique identifier for the server’s IP.
