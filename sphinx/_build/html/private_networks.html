list_private_networks(page=None, per_page=None, sort=None, q=None, fields=None)¶

Returns a list of all private networks.

Parameters:
  • page (int) – Allows the use of pagination. Indicate which page to start on.
  • per_page (int) – Number of items per page.
  • sort (str) – sort='name' retrieves a list of elements sorted alphabetically. sort='creation_date' retrieves a list of elements sorted by their creation date in descending order.
  • q (str) – q is for query. Use this parameter to return only the items that match your search query.
  • fields (str) – Returns only the parameters requested. (i.e. fields=’id, name, description, hardware.ram’)

The “PrivateNetwork” Class¶

class PrivateNetwork(name=None, description=None, network_address=None, subnet_mask=None)¶

Pass a PrivateNetwork instance into the create_private_network() method to create a private network. There are also a few helper methods available to perform simple requests after creating your private network.

Parameters:
  • name (str) – private network name.
  • description (str) – private network description.
  • network_address (str) – Private network address. (valid IP)
  • subnet_mask (str) – Subnet mask (valid subnet for the given IP).

Methods:

get()¶

Retrieves the private network’s current specs.

servers()¶

Returns a list of the servers attached to the private network.

wait_for()¶

Polls the PrivateNetwork resource until an ACTIVE, POWERED_ON, or POWERED_OFF state is returned.

The “AttachServer” Class¶

class AttachServer(server_id=None, rights=None, server_ip_id=None)¶

Use the AttachServer class to attach servers or server IPs to various containers.

Parameters:
  • server_id (str) – the unique identifier for the server.
  • rights (str) – server rights for accessing a shared storage. Possible values are 'R' or 'RW'.
  • server_ip_id (str) – the unique identifier for the server’s IP.
